Percy Agentic Workflow

A multi-agent system for Claude Code that takes a problem statement through research, product definition, design exploration, implementation, QA verification, and PR creation — with mandatory user approval gates at every stage.

20 agents 17 commands 8 skills 8 stages 9 approval gates

Install

Quick Setup

  1. Get the package
    Clone or download this repo
    git clone https://github.com/Peter202112/percy-agentic-workflow.git
  2. Go to your Percy monorepo
    The installer copies files into the current directory
    cd /path/to/percy
  3. Run the installer
    One command sets up everything
    /path/to/percy-claude-workflow/install.sh --all

What It Does

Provide a problem statement or PRD. The gated orchestrator walks you through 8 stages with 9 approval gates. Every stage pauses for your approval — feedback loops back to re-run that stage.

1

Research

Competitive analysis, codebase mapping, 3 solution directions. Includes User Benefit Validation: problem solved, impact metric, alignment check.

percy-researcher + code-explorer reviews/research-{feature}.html 🚦 Gate 1
2

Product Brief

Personas, phased deliverables, acceptance criteria, edge cases, success metrics.

percy-pm reviews/product-brief-{feature}.html 🚦 Gate 2
3

Design MANDATORY

Blocks until Figma page URL + design system reference provided. Creates 3+ screens using extracted tokens. After approval, mockups are embedded into product brief (single source of truth).

percy-design-explorer + Figma MCP Figma + updated product brief 🚦 Gate 3
4

Implementation Plan

DB schema, API models/services/controllers, Web components, file-by-file change list. Uses product brief with embedded mockups.

code-architect 🚦 Gate 4
5

Implementation (API + Web)

API: migration → model → service → controller. Web: model → adapter → component. ESLint auto-fix. Two sub-gates for API and Web diffs.

Direct code writing 🚦 Gate 5a (API) 🚦 Gate 5b (Web)
6

QA Verification

Docker setup, API service specs, API request specs, frontend build, ESLint. Produces comprehensive test report with detailed test plan, suite, and test cases (objective, steps, data, expected/actual, notes).

/percy-qa reviews/testresults.md + .html 🚦 Gate 6
7

Commit & PR

Commit in both repos with descriptive messages. Push and create PR only after explicit approval.

git + gh pr create 🚦 Gate 7
8

Cleanup

Presents running resources (Docker, dev server, browser). Warns restart takes 2-5 min. Options: shut down all, keep Docker, keep everything. Default: keep running.

🚦 Gate 8

Where Agents Live (and Why)

Claude Code scopes agent discovery by working directory. This keeps each context clean — you only see relevant agents.

LocationAvailable WhenContains
.claude/agents/ Always — from any directory Workflow agents (orchestrator, researcher, PM, design-explorer, usability-reviewer) and cross-repo agents (contract-reviewer)
percy-web/.claude/ Only in percy-web/ Web review agents (design-system, component, ember-data, template, accessibility), web commands, web PR validation
percy-api/.claude/ Only in percy-api/ API review agents (endpoint, job, spec, security, migration, etc.), API commands, API PR validation
Rule of thumb: If an agent reads/writes code in only one repo, it lives in that repo's .claude/. If it spans both repos or operates at the product level (research, design, usability), it lives at the root.
