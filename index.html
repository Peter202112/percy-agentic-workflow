Percy Agentic Workflow

A multi-agent system for Claude Code that takes a problem statement through research, product definition, design exploration, implementation, and validation.

20 agents 17 commands 5 skills 9 phases 6 review gates

Install

Quick Setup

  1. Get the package
    Clone or download this repo
    git clone git@github.com:your-org/percy-claude-workflow.git
  2. Go to your Percy monorepo
    The installer copies files into the current directory
    cd /path/to/percy
  3. Run the installer
    One command sets up everything
    /path/to/percy-claude-workflow/install.sh --all

What It Does

Provide a problem statement or PRD. The orchestrator walks you through 9 phases, producing an HTML review report at each gate.

1

Research

Competitive analysis (Applitools, Chromatic, etc.), problem decomposition, 3 solution directions.

@percy-researcher reviews/research-{date}.html Review Gate
2

Product Brief

Reframes problem, defines user personas, features with acceptance criteria, implementation phases, success metrics.

@percy-pm reviews/product-brief-{date}.html Review Gate
3

Design Exploration

3 UI approaches mocked up in Figma using Percy’s component library. Comparison matrix and recommendation.

@percy-design-explorer Figma file + HTML report Review Gate
4

Implementation Plan

File-level change list, API contract mapping, dependency order.

@percy-orchestrator reviews/implementation-plan-{date}.html Review Gate
5

API Implementation

Migration → model → service → controller → serializer → specs. Spot-checked by review agents.

percy-api agents
6

Web Implementation

Model → adapter → route → components → templates. Parallel after serializer is defined.

percy-web agents
7

Validation

14+ agents check code quality, security, accessibility, design tokens, API contracts.

percy-pr-validate reviews/validation-{date}.html
8

Usability Review

Nielsen’s 10 heuristics. Cross-referenced against PRD, competitor research, and chosen design.

@percy-usability-reviewer reviews/usability-review-{date}.html Review Gate
9

CI + PR

Buildkite CI monitoring, PR creation with summary, contract alignment, and report links.

reviews/pr-summary-{date}.html

Where Agents Live (and Why)

Claude Code scopes agent discovery by working directory. This keeps each context clean — you only see relevant agents.

LocationAvailable WhenContains
.claude/agents/ Always — from any directory Workflow agents (orchestrator, researcher, PM, design-explorer, usability-reviewer) and cross-repo agents (contract-reviewer)
percy-web/.claude/ Only in percy-web/ Web review agents (design-system, component, ember-data, template, accessibility), web commands, web PR validation
percy-api/.claude/ Only in percy-api/ API review agents (endpoint, job, spec, security, migration, etc.), API commands, API PR validation
Rule of thumb: If an agent reads/writes code in only one repo, it lives in that repo's .claude/. If it spans both repos or operates at the product level (research, design, usability), it lives at the root.
