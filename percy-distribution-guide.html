Distribution Guide

How to share the Percy Agentic Workflow with your team and other projects.

How It Works

The Percy agentic workflow is pure markdown files — no npm packages, no compiled code, no runtime dependencies. Claude Code automatically discovers agents, skills, and commands from .claude/ directories. Distribution is file copying.

Key insight: Anyone who has the .claude/ files in their repo can use /percy-orchestrator-gated, /percy:endpoint-safety, etc. immediately. No installation step, no configuration.

Distribution Options

Option 1: Commit to the Repo Recommended

Best for: Percy team members working in the monorepo

Commit the .claude/ directories to the Percy monorepo. Every developer who clones the repo gets the agents automatically.

git add .claude/ percy-web/.claude/ percy-api/.claude/ reviews/.gitkeep
git commit -m "Add Percy agentic workflow for Claude Code"
git push

Pros

  • Zero setup for teammates
  • Agents evolve with the codebase
  • PRs can update agents alongside code

Cons

  • Repo-specific, not reusable elsewhere

Option 2: Setup Script Flexible

Best for: Installing into forks, branches, or fresh clones

Use the included setup.sh to copy agents selectively:

# Install everything
.claude/setup.sh --all

# Into a different repo
.claude/setup.sh --all --target /path/to/other-repo

# Just the workflow agents (research, PM, design, orchestrator)
.claude/setup.sh --workflow

# Just the code review agents
.claude/setup.sh --web-reviewers
.claude/setup.sh --api-reviewers

# Interactive mode — pick and choose
.claude/setup.sh

Pros

  • Install only what you need
  • Works across repo copies
  • Interactive or scripted

Cons

  • Manual re-run on updates

Option 3: Template Repo Portable

Best for: Sharing across teams, orgs, or open-sourcing

Extract the setup into its own Git repo:

# Create the template repo
mkdir percy-claude-agents && cd percy-claude-agents && git init

# Copy agent definitions
cp -r /path/to/Percy/.claude .
cp -r /path/to/Percy/percy-web/.claude percy-web/.claude
cp -r /path/to/Percy/percy-api/.claude percy-api/.claude

git add -A && git commit -m "Percy agentic workflow template"
git remote add origin git@github.com:your-org/percy-claude-agents.git
git push -u origin main

Consumers clone and run setup:

git clone git@github.com:your-org/percy-claude-agents.git /tmp/percy-agents
/tmp/percy-agents/.claude/setup.sh --all --target /path/to/their-repo

Pros

  • Versioned independently (tag releases)
  • Shareable across teams/orgs
  • Can publish as open-source

Cons

  • Extra repo to maintain
  • Consumers re-run setup on updates

Option 4: User-Level Install Global

Best for: Personal use across all projects

Install to ~/.claude/ so agents are available everywhere:

# Workflow agents
cp -r .claude/agents/* ~/.claude/agents/

# Skills
cp -r .claude/skills/* ~/.claude/skills/

Pros

  • Available in every Claude Code session
  • No per-repo setup

Cons

  • Percy-specific agents may not fit other projects
  • No per-project customization

Understanding Agent Scoping

Claude Code scopes agent discovery by working directory. This is why agents live in different .claude/ directories across the monorepo.

Rule of thumb: If an agent reads/writes code in only one repo, it lives in that repo's .claude/. If it spans both repos or operates at the product level (research, design, usability), it lives at the root.
LocationAvailable WhenWhat Goes Here
.claude/ (root) Always — from any directory in the monorepo Workflow agents (orchestrator, researcher, PM, design-explorer), cross-repo agents (contract reviewer, usability reviewer), shared skills (report generator)
percy-web/.claude/ Only when cd percy-web Web-specific reviewers (design-system, component, template, ember-data, accessibility), web commands, web PR validation
percy-api/.claude/ Only when cd percy-api API-specific reviewers (endpoint, job, spec, security, migration, etc.), API commands, API PR validation

This means when distributing, the directory structure matters. The setup script preserves the correct structure automatically. If copying manually, ensure agents end up in the right .claude/ directory.

Customization

Everything is plain markdown — edit directly, no build step needed.

Modify Existing Rules

Open any agent .md file, edit the rules section. Changes take effect on the next invocation.

Add a New Agent

Create a .md file in .claude/agents/:

---
name: my-custom-agent
description: "What this agent does and when to invoke it."
---

Your agent prompt here...

Invoke with @my-custom-agent in Claude Code.

Add a New Command

Create a .md file in .claude/commands/percy/:

---
description: What this command does
allowed-tools: Read, Glob, Grep, Bash
---

# Command Name

Instructions...

Invoke with /percy:command-name.

Override Precedence

Project-level .claude/ overrides user-level ~/.claude/. Use this for project-specific customizations.
